# 유클리드 호제법 (Euclidean algorithm)
최소공배수와 최대공약수를 구할 때 사용할 수 있는 방법이다.

## 정의
두 양의 정수 혹은 두 다항식의 **최대공약수(greatest common divisor, GCD)** 값을 구하는 방법이다.   
'서로(互) 나누다(除)'라는 의미를 가진 방식으로서, 다음과 같이 설명된다.
> 두 양의 정수 $$a, b (a > b)$$에 대하여, $$a = bq + r (0 ≤ r < b)$$라고 한다면, $$gcd(a, b) = gcd(b, r)$$을 만족하며
$$r = 0$$일 때 $$a, b$$의 최대공약수는 $$b$$가 된다.

## 증명 방법
$$gcd(a, b) = x$$로 두고 $$gcd(b, r) = x'$$이라고 두자.   
그러면 적당한 서로수인 정수 A와 B에 대하여, $$a = Ax$$와 $$b = Bx$$가 성립한다.   
이를 주어진 식 $$a = bq + r$$에 대입하면, $$Ax = Bxq + r$$에서 $$r = x(A - Bq)$$이다.   
이때 $$b$$와 $$r$$ 모두 $$x$$에 대한 식으로 정리되므로 $$x$$는 $$b$$와 $$r$$의 공약수임을 알 수 있다.   
$$x'$$의 약수가 $$x$$임을 알 수 있다.   
$$x' = mx$$라고 둔다면 적당한 서로소인 두 정수 $$k, l$$에 대하여   
$$Bx = x'k = mxk$$와 $$x(A - Bq) = x'l = xml$$라는 두 식이 성립한다.
각각 변을 x로 나누게 된다면 $$B = mk, A - Bq = ml$$이 된다.
여기서 $$A = Bq + ml = mkq + ml = m(kq + l)$$임에 따라 $$m$$은 $$A$$와 $$B$$의 공약수이다.
서로소인 $$A, B$$에 대하여 $$gcd(A, B) = 1$$이므로 항상 $$m = 1$$이고 $$x' = mx = x$$이다.

## 수를 활용한 확인
두 수의 최대공약수를 구하기 위해 사용할 수 있다 했으니, $$gcd(208, 124)$$를 구해보자.
1. $$208 = 124 × 1 + 84$$이므로 $$gcd(208, 124) = gcd(124, 84)$$
2. $$124 = 84 × 1 + 40$$이므로 $$gcd(124, 84) = gcd(84, 40)$$
3. $$84 = 40 × 1 + 44$$이므로 $$gcd(84, 40) = gcd(40, 44)$$ / 큰 수를 왼쪽으로 보내기 위해 $$gcd(44, 40)$$
4. $$44 = 40 × 1 + 4$$이므로 $$gcd(44, 40)  = gcd(40, 4)$$
5. $$40 = 4 × 10 + 0$$이므로 $$r = 0$$임에 따라 $$gcd(208, 124) = gcd(40, 4) = 4$$로 최대공약수는 $$4$$

## 대략적인 코드화
1. 재귀함수를 활용하는 경우
```JAVA
int Euclidean(int a, int b) {
  if(b == 0) return a;
  return Euclidean(b, a % b);
}
```

2. 반복문을 활용하는 경우
```JAVA
int Euclidean(int a, inㅅ b) {
  while(b > 0) {
    int store_b = b;
    b = a % b;
    a = store_b;
  }
  return a;
}
```

## 시간 복잡도와 효율성
나눗셈 횟수는 $$O(\log{a})$$로 나타나며 1회의 복잡도가 제수와 몫의 자리수 개수에 비례한다는, 즉 $$O(\log{a} \log{(a/b)})$$로
나타나는 것까지 고려한다면 유클리드 호제법 전체의 시간 복잡도가 $$O((\log {a})^2)$$로 나타나는 것도 보일 수 있다.

1. 유클리드 호제법의 기본 과정 속에서 a와 b의 값이 계속해서 줄어드는 과정을 따르므로 최악의 경우 나누는 횟수는 $$O(\log{min(a, b)})$$이다.
2. 나눗셈 과정에서 a/b에 대해 제수 b의 크기는 $$O(\log {b})$$이고 몫의 자릿수는 $$O(\log{(a/b)})$$이므로 나눗셈 한 번에 $$O(\log a \log(a/b))$$이다.
3. 결국, 최악의 경우 나누는 횟수가 $$O(\log a)$$임에 따라 각 나눗셈의 비용을 활용하여 전체 시간 복잡도의 추정이 가능하다.
4. 첫 번째 나눗셈은 $$O(\log {a} \log{(a/b)})$$이고, 두 번째 나눗셈부터는 크기가 점점 더 작아지며 반복된다.
5. 각 나눗셈의 비용 $$O(\log {a} \log{(a/b)})$$가 나누는 횟수 $$O(\log {a})$$에 대해 발생하므로 전체 시간 복잡도는 $$O({(\log {a})}^2)$$임을 확인할 수 있다.

## 최소공배수로의 확장
최소공배수(least common multiple, LCM)의 경우 $$lcm(a, b) = \frac{|ab|}{gcd(a, b)}$$가 성립하므로 두 수의 곱을 최대공약수로 나누면 된다.
